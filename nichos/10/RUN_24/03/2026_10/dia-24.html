Lista salvável + prova social + oferta (roteiro reproduzível)

Use este template para qualquer tema (nutrição, treino, skincare, finanças, etc.) • 10 slides (funil completo em 1 post)

Resumo

Objetivo: salvar → confiar → agir
Promessa
Entregar uma curadoria pronta (por categorias) com 1 métrica-chave por item
Retenção
Padrão repetido + progressão por categorias + “save this”
Conversão
Sequência de provas + slide final com CTA único
Ideia central: “Dá para atingir {objetivo} sem abrir mão de {desejo}”, com exemplos práticos e prova de que funciona.

Copiar roteiro

Edite os campos entre chaves { } e mantenha a estrutura para repetir com outros temas.

Estrutura

Macro: educativo → confirmação → ação
Bloco
1) Capa (promessa + micro-CTA) → 2) Listas por categoria (salvável) → 3) Provas sociais em sequência (confiança) → 4) Oferta (próximo passo)
Por que funciona
O usuário ganha um “arquivo de bolso” (lista), depois vê evidência real (prints) e, quando está convencido, recebe um caminho simples (CTA único).
Pontos não negociáveis
Repetição de layout nos slides 2–5; 1 dado-chave por item; provas com headline; oferta com poucos bullets e um CTA.

Roteiro

Slide a slide (pronto para preencher)

Slides sugeridos (1..10)

  1. Slide 1 — Capa: “{Ideias/Opções} de {X} para {Objetivo}” + “Salve este post →”. (Imagem de contexto real.)
  2. Slide 2 — Categoria 1: “- {cat1} -” + 4 itens (grid 2x2) com métrica principal sublinhada.
  3. Slide 3 — Categoria 2: “- {cat2} -” + 4 itens (mesmo padrão).
  4. Slide 4 — Categoria 3: “- {cat3} -” + 4 itens (mesmo padrão).
  5. Slide 5 — Categoria 4: “- {cat4} -” + 4 itens (mesmo padrão) + rodapé técnico.
  6. Slide 6 — Prova social 1: print + tarja: “{resultado} em {tempo}”.
  7. Slide 7 — Prova social 2: print + tarja: “{benefício percebido}”.
  8. Slide 8 — Prova social 3: print + tarja: “{objeção quebrada}”.
  9. Slide 9 — Prova social 4: print + tarja: “{consistência/rotina/resultado}”.
  10. Slide 10 — Oferta: “Incluído em {serviço}” + 4–6 bullets + CTA único (DM com palavra-chave ou link).
Atalho de produção: defina 1 métrica (ex.: proteína, fibras, carbo, sódio, custo, tempo de preparo) e use a mesma em todos os itens. Consistência melhora compreensão e “salvabilidade”.

Elementos visuais

Direção de arte replicável
  • Fundo clean nos slides de lista (para leitura rápida) + cor emprestada das embalagens/prints.
  • Header repetido no topo (nome da série/tema) para criar padrão.
  • Subcategoria em itálico com hífens: “- {categoria} -”.
  • Grid 2x2 com imagens recortadas + legendas padronizadas.
  • Sublinhar o dado-chave (a métrica principal) em todos os itens.
  • Prova social: tarja preta (headline curta) + marca-texto em 2–3 trechos do print.
  • Oferta: mockup do app/planilha/consultoria + bullets escaneáveis.

Copy e ganchos

Fórmulas substituíveis
Título (capa)
“{Solução simples e desejável} para {Objetivo}”
Alternativas: “{X} opções de {coisa} para {situação} (salve!)” • “{Categoria} que parece {benefício} e cabe em {restrição}”
Gancho (legenda)
“Você não precisa {abrir mão de X / sofrer Y} para {objetivo}.”
“Se {dor}, escolha {critério} e use essas categorias como mapa.”
Rodapé técnico
“Dica: combine com {proteína/fibra/gordura} para {saciedade/energia/glicemia}.”
“Ajuste por {rotina/treino/medicação}. Se tiver condição clínica, personalize.”
Tarjas (provas)
“{X} em {Y}” • “Menos {sintoma}” • “Mais {benefício}” • “Consistência sem {objeção}”

CTA

1 primário + 1 secundário
  • CTA primário (DM): “Comente/mande {PALAVRA} que eu te envio {bônus/lista/guia}.”
  • CTA primário (Link): “Link na bio para {entrar na lista / baixar o guia / aplicar}.”
  • CTA secundário (salvar): “Salve para consultar quando estiver {no mercado/planejando a semana}.”
  • CTA secundário (compartilhar): “Envie para alguém que sempre pergunta sobre {tema}.”
Regra: no slide 10, use apenas 1 CTA. Se quiser dois, deixe o secundário para a legenda.

Checklist

Criação sem furos
  • [ ] Capa com promessa simples (até 10 palavras) + “salve este post”.
  • [ ] 3–4 slides com o mesmo layout (não reinventar a cada página).
  • [ ] Cada item tem 1 métrica principal sublinhada (o motivo de estar na lista).
  • [ ] Rodapé técnico curto (1 linha) para credibilidade/nuance.
  • [ ] 2–4 provas sociais anonimizadas + headline forte.
  • [ ] Slide final com 4–6 benefícios + mockup + CTA único.
  • [ ] Legenda com: gancho (1–2 linhas) + como usar + CTA.

Variações

Adaptações com o mesmo impacto
Troque “snacks” por
“café da manhã”, “jantar rápido”, “marmitas”, “lancheira infantil”, “pré/pós-treino”, “opções no iFood”, “combinações para viagem”.
Troque a métrica
Proteína → fibras → açúcar adicionado → sódio → calorias → custo por porção → tempo de preparo → índice de saciedade (escala simples 1–5) → densidade nutricional (critério próprio).
Troque “prova social” por
Prints → resultados de exames (com ética e contexto) → check-ins de consistência → notas de alunos → avaliações públicas → “antes/depois” de hábitos (não só corpo).
Troque “oferta” por
Consultoria 1:1 → grupo/mentoria → e-book → aula → checklist PDF → desafio de 7 dias → lista de espera; mantendo: bullets claros + 1 CTA.
Versão para outro nicho
Skincare: “produtos por categoria (manhã/noite/pele oleosa)” + prova (feedback) + oferta (consulta).
Finanças: “apps/hábitos por categoria (controle, investimento, dívidas)” + prova (relatos) + oferta (mentoria).
Dica de consistência: salve este HTML como modelo do seu “Dia 24” e só substitua {tema, categorias, métricas, provas e CTA} em cada repetição.
