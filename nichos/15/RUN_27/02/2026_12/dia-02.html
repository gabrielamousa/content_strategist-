Resumo

Template universal para Reels de 25–45s que performa bem por combinar: confronto + promessa ética/qualitativa + prova com números + sequência de “sacadas” + pergunta final.

Confronto (polariza) Números cedo (credibilidade) Insights em metralhadora Visual guiado (setas/zoom) Pergunta (comentários)

Estrutura (reproduzível e neutra)

  1. 0–3s — Hook confronto: chame o “vilão comum” do seu nicho.
  2. 3–6s — Tese: “dá para ter resultado sem fazer X”.
  3. 6–12s — Prova rápida: 2–3 números + contexto (caso real, teste, cliente, exemplo, estudo).
  4. 12–32s — 3 a 5 sacadas: 1 ideia por frase, com ordem lógica (estratégia → execução → resultado).
  5. 32–40s — Regra/alerta: “combinado? não faça X achando que é o caminho”.
  6. 40–45s — Pergunta: convide a comparar a realidade da audiência (“aí é assim ou diferente?”).
  7. (Opcional) Últimos 1–2s — Referência: nome do case/fonte/metodologia (segura retenção).

Roteiro (pronto para gravar)

Preencha os colchetes e grave com cortes rápidos. Ideal: voz em off + imagens do “case” (prints, tela, before/after, foto, demo, protótipo).

HOOK (0–3s) “Se você tá pensando em [ATAlHO/JEITINHO/PRÁTICA RUIM] pra conseguir [RESULTADO], para e olha isso.” TESE (3–6s) “Pra ter [RESULTADO], você não precisa [COISA FEIA/ANTIÉTICA/DE BAIXA QUALIDADE]. Dá pra fazer melhor.” PROVA COM NÚMEROS (6–12s) “No caso de [CASE REAL], os números foram: [NÚMERO 1], [NÚMERO 2] e [NÚMERO 3]. E ainda teve [DIFERENCIAL].” SACADAS (12–32s) — escolha 3 a 5 “SACADA #1: [ESTRATÉGIA PRINCIPAL] — isso evita [PROBLEMA].” “SACADA #2: [SIMPLIFICAÇÃO/PROCESSO] — deixa [MAIS RÁPIDO/BARATO/ESCALÁVEL].” “SACADA #3: [QUALIDADE/EXPERIÊNCIA] — melhora [CONFORTO/USO/RESULTADO PERCEBIDO].” “SACADA #4: [CONFORMIDADE/REGRA/RESTRIÇÃO] — encaixa em [NORMA/POLÍTICA/LIMITAÇÃO] sem gambiarra.” “SACADA #5: [APRESENTAÇÃO/EMBALAGEM] — aumenta [CONVERSÃO/VALOR/RETENÇÃO].” REGRA + ALERTA (32–40s) “Regra simples: se o seu plano depende de [PRÁTICA RUIM], ele não é estratégia — é dívida com o futuro. Combinado?” PERGUNTA (40–45s) “Aí onde você tá, o padrão é [VERSÃO BOA] ou ainda é [VERSÃO RUIM]? Comenta.” REFERÊNCIA (opcional, 1–2s) “Case: [NOME/CLIENTE/FONTE] (ano/resultado).”

Elementos visuais (alto impacto + fácil de replicar)

Ordem de B-roll (lógica):

  1. “Problema” (ex.: exemplo ruim / tela com erro / antes)
  2. “Prova” (ex.: gráfico / números / print / exemplo real)
  3. Sacada #1 (ex.: esquema simples com seta)
  4. Sacada #2 (ex.: passo na tela / checklist rápido)
  5. Sacada #3 (ex.: resultado / depois / depoimento)
  6. Fecho com pergunta em tela cheia

Padrão de edição:

  • Cortes a cada 0,7–1,5s nas sacadas.
  • Zoom + highlight (círculo, seta, underline) no ponto que você cita.
  • Texto na tela: palavra-chave + número (sem legenda longa).
  • SFX leve (whoosh/click) + música discreta (voz manda).
  • Tela final 1–2s: pergunta + CTA.

Copy e ganchos (neutros)

Ganchos prontos (escolha 1):

  • Confronto: “Se você faz [X] pra conseguir [Y], você tá perdendo [Z].”
  • Mito vs verdade: “Mito: pra ter [Y], precisa [X]. Verdade: dá pra fazer com [alternativa].”
  • Ética/qualidade: “Resultado não é desculpa pra entregar [versão ruim].”
  • Número chamativo: “Com [RECURSO LIMITADO], deu pra alcançar [RESULTADO] — sem [atalho].”

Legenda curta (modelo):

“Não é sobre fazer mais. É sobre fazer melhor. No case [X], 3 decisões mudaram o jogo: [1], [2], [3]. Pergunta: aí o padrão é [bom] ou [ruim]? (Se quiser, eu adapto esse roteiro pro seu cenário: comenta ‘QUERO’ + seu contexto.)”

CTA (escolha 1–2)

  • Comentário (distribuição): “Aí é mais [bom] ou [ruim]? Comenta.”
  • Salvar (recorrência): “Salva pra usar quando for [tomar decisão/fazer projeto/planejar].”
  • Compartilhar: “Manda pra alguém que ainda acha que [atalho] é estratégia.”
  • Seguir: “Segue pra mais exemplos práticos com números + sacadas rápidas.”
  • Oferta leve: “Quer que eu analise seu caso? Comenta [palavra] + [dado].”

Checklist de criação (do zero ao post)

Objetivo: tornar o formato replicável em qualquer nicho, com consistência de retenção.

Defina o “vilão” do nicho: a prática ruim que todo mundo conhece.
Escolha 1 case real (seu, de cliente, público, estudo) e valide o que pode citar.
Separe 2–3 números (tempo, custo, volume, taxa, antes/depois).
Liste 3–5 sacadas (estratégia → execução → resultado). 1 frase cada.
Roteiro com frases curtas e sem explicação longa (puxe pro visual).
Adiante os números até 12s (credibilidade cedo).
Prepare 6–10 clipes: problema, prova, cada sacada, resultado, tela final.
Edição: cortes a cada ~1s nas sacadas + texto curto sincronizado.
Tela final: pergunta + CTA (comentário ou salvar).
Fixe um comentário seu: “Quer adaptação pro seu caso? diga [X] + contexto.”

Variações (3 formatos do mesmo tema)

  1. Variação “3 erros”: Hook: “Se você tá fazendo X, Y e Z…” → depois mostre a correção com números.
  2. Variação “antes/depois”: Comece pelo resultado final e volte explicando 3 sacadas que levaram até lá.
  3. Variação “mini-auditoria”: Pegue um exemplo comum (público) e aplique seu checklist em 30s.

Banco de palavras (neutras) para substituir “caixote” sem perder impacto:

[versão ruim] = gambiarra / atalho / solução preguiçosa / cópia mal feita / remendo / “parece bom mas não é” [versão boa] = solução bem pensada / justa / eficiente / escalável / de qualidade / sustentável

Dica de UX: mantenha 1 ideia por frase e deixe o visual “provar” o que você diz. Se a frase precisa de parágrafo, virou outro vídeo.
