8 ideias de FONTES para TÍTULOS (modelo reproduzível)

Post original: https://www.instagram.com/p/DVtcMPtjTiF/

Entrega visual rápida + micro-CTA no meio + CTA por palavra-chave

Ações rápidas

Abrir post original
Objetivo Salvar + Comentários + DM

Dica: mantenha o mesmo layout nos itens (slides de entrega) e mude apenas o conteúdo principal. Isso cria ritmo e “colecionabilidade”.

Parâmetros editáveis

  • X para YTemplate de título

    Ex.: “FONTES para TÍTULOS”, “CORES para MARCAS”, “LAYOUTS para PORTFÓLIO”.

  • 5 itens + 1 pausaRitmo

    3 itens → slide “Salva” → 2 itens → CTA final.

Resumo

Modelo de carrossel de curadoria com estética “typographic poster”: promete um resultado na capa, entrega itens em tipografia gigante (consumo rápido) e intercala um micro-CTA de salvar antes de fechar com CTA de comentário por palavra-chave.

Quando usarIdeal

Quando você quer virar referência (curadoria), gerar salvamentos e iniciar conversa no direct sem precisar explicar muito.

O que torna reproduzívelSistema

Layout fixo + conteúdo variável. A repetição reduz atrito e a variação entre itens mantém interesse.

Estrutura

  • Formato9:16

    Preferencial 9:16 (Reels/Stories). Dá para adaptar para 1080×1350 mantendo margens de segurança de 6–8%.

  • Número de slides6–10

    Recomendação: 8 slides (equilíbrio entre entrega e retenção).

  • Ritmo do carrosselPadrão

    1) Capa/Promessa → 2–4) Itens → 5) “Salva” → 6–7) Itens → 8) CTA final.

  • Regra de ouroClareza

    1 ideia por slide. Texto do item: 1–3 palavras no máximo. Legenda: 1 linha.

Slides sugeridos (1..8)

  • Slide 1 — Capa (promessa)Hook

    [RECURSO/OBJETO] PARA [USO/RESULTADO]
    Ex.: “FONTES PARA TÍTULOS”. Subtexto opcional: “Desliza →”.

  • Slide 2 — Item 1Entrega

    H1 gigante: [ITEM]
    Legenda pequena: [NOME / ONDE USAR]

  • Slide 3 — Item 2Entrega

    Repete layout. Troca apenas o item.

  • Slide 4 — Item 3 (com variação)Variação

    Mesmo layout, mas aplique 1 “twist” visual (textura, outline, sombra, distorção leve) para aumentar retenção.

  • Slide 5 — Pausa (micro-CTA)Retenção

    Frase curta: “Salva pra usar depois.” / “Guarda isso.” + ícone grande (seta/círculo) para respirar e reforçar o salvar.

  • Slide 6 — Item 4Entrega

    Volta ao padrão dos itens.

  • Slide 7 — Item 5Entrega

    Último item com o maior “wow” (o mais forte/mais útil).

  • Slide 8 — CTA finalConversão

    Pergunta + instrução: “Quer o pack completo de [TEMA]? Comenta ‘[PALAVRA]’ que eu te mando no direct.”

Roteiro

Use o texto abaixo como base (substitua os campos entre colchetes). Mantém o mesmo impacto em qualquer tema.

Personalize: troque apenas o tema, os 5 itens e a palavra-chave. O resto (estrutura e ritmo) permanece.

Elementos visuais

  • IdentidadeHigh-tech

    Fundo escuro + alto contraste. Use 2 cores de apoio e uma cor de destaque (pode ser aplicada em gradiente).

  • Componentes fixosConsistência

    Assinatura (canto inferior esquerdo) + seta de navegação (inferior direito) em todos os slides.

  • HierarquiaTipografia

    H1 gigante ocupando 60–80% da área útil. Legenda 1 linha. Microtexto só em CTAs.

  • Grid e margensSegurança

    Margem de segurança: 6–8% (evita cortes). Conteúdo central (80–90% da largura).

  • Padrão por tipo de slideRitmo

    Mesmo fundo para slides de itens; outro fundo (mais forte) para “Salva” e CTA final.

  • Variação controladaRetenção

    Só 1 slide com efeito especial (ex.: outline/textura) para criar “pico” sem virar bagunça.

CTA

  • CTA principal (comentário)Baixa fricção

    Estrutura: Pergunta de qualificação + oferta + ação simples + retorno.
    Modelo: “Quer o [entregável] de [tema]? Comenta ‘[PALAVRA]’ que eu te mando no direct.”

  • Micro-CTA (salvar)Meio do carrossel

    Use no slide 5 para capturar alta intenção: “Salva pra usar depois.” (curto, imperativo e óbvio).

Checklist

  • Pré-produção10–20 min

    [ ] Definir “X para Y” (tema + resultado) [ ] Selecionar 5 itens (ordem: bom → melhor → melhor ainda) [ ] Definir palavra-chave do CTA [ ] Definir entregável (opcional): lista completa, pack, template

  • DesignSistema

    [ ] Criar 3 layouts-base: Capa / Item / CTA [ ] Fixar assinatura (inf. esq.) e seta (inf. dir.) [ ] Definir 3 níveis: H1 gigante / legenda / microtexto [ ] Margens de segurança 6–8%

  • RevisãoQualidade

    [ ] Cada slide tem 1 ideia? [ ] H1 com no máximo 2–3 palavras? [ ] Legibilidade alta em tela pequena? [ ] Slide 4 tem variação (sem virar ruído)? [ ] Slide 5 “salva” está no meio (não no final)? [ ] CTA final com palavra-chave destacada?

  • PublicaçãoOperação

    [ ] Legenda com gancho na 1ª linha [ ] Pin de comentário: “Comenta ‘[PALAVRA]’” [ ] Responder comentários rápido (primeira hora) [ ] Automatizar DM (se usar ferramenta) ou enviar manualmente

Variações

  • Mesma estrutura, outros temasSubstituições

    - “FONTES para TÍTULOS” → “PALETAS para MARCAS” - “FONTES para TÍTULOS” → “GRIDS para EDITORIAIS” - “FONTES para TÍTULOS” → “EFEITOS para POSTS” - “FONTES para TÍTULOS” → “LAYOUTS para PORTFÓLIO”

  • Se quiser aumentar retenção+1 slide

    Adicione um slide extra entre 6 e 7: “Bônus: meu favorito” ou “Erro comum ao usar [tema]” (mantém o usuário até a CTA).

  • Se quiser reduzir para 6 slidesEnxuto

    Faça 1 capa + 3 itens + 1 “salva” + 1 CTA final. Mantém a lógica sem perder o ritmo.

  • Trocas de CTA (mesmo efeito)Alternativas

    - “Comenta ‘[PALAVRA]’…” - “Manda ‘[PALAVRA]’ no direct…” - “Quer que eu te envie? Escreve ‘[PALAVRA]’ aqui embaixo.”
